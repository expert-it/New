Room 1003 - 11:20 AM to 12:20 PM
Creating a 360 degree view of a user.

This talk turns to an important problem that many large companies try to solve after they engineer a big data platform -- tracking a user across their portfolio of products, also called "creating a 360 degree view of a user". Since many of these companies did not plan to integrate their user data a-priori, either because they optimized their products independently until now, or because they have grown by acquisition, the first challenge is usually to "wrangle" or unify dimensions on which to describe the user, as well as develop the common language (the verbs, the nouns, the adjectives, and the adverbs) to describe user actions. Often, the user ID can only be matched probabilistically across products. Such a unification also requires constant attention as products go independently through releases and changes in event logs. This talk will describe the problems and solutions taken to create a 360 degree view for the user of products from a major fortune 100 company.

The challenges of building Data Lakes and Data Applications on Hadoop.

As the Hadoop ecosystem matures, and the projects that comprise it expand and diverge over time, there is a significant impact on how data application architectures are designed. Part of any technology becoming mainstream is evolving the discussion from what a technology is to how it is used and the business impact it has, Unfortunately, this progression is hindered by the increasing diversity in the Hadoop ecosystem.
Traditional web infrastructure went through this transition over the past 20 years, from client-server, to app server, to a focus on user experience and business value. Hadoop is going through this transformation now and we are at the stage when value and user experience matter the most -- businesses want Hadoop technologies to create data-driven business value, not just be a cheap way to store and process data.
In this talk, we will discuss the challenges that the industry is currently facing and present the unified data application architecture approach that the Cask Data Application Platform (CDAP) takes to address these issues.. CDAP aims to bring simplicity, productivity and an enhanced user experience to Hadoop – none of which have been a primary focus in this ecosystem to date. We will also discuss real world use-cases in the enterprises where CDAP is used in production with great success.

Room 1004 - 1:45 PM to 2:45 PM
Detecting Anomalous User Behavior within a Large Enterprise

Insider threat detection and malware presence detection inside an enterprise network are two issues that receive considerable attention within information security departments of large enterprises. In this talk, we introduce a 2-stage anomaly detection framework that creates alerts for forensic teams to address both of these issues. The input data source for this framework is the raw active directory logs capturing each employee's server access pattern over a period of year or so. The first stage of the anomaly detection framework is based on PCA (Principal Components Analysis) of access pattern data and is a user-specific behavioral model. It flags users whose current weekly server access pattern is sufficiently different from his/her own server access pattern in the past. The second stage of the anomaly detection framework exploits the server-access pattern of a user's peers as well as the server's peers to reduce false positives. Specifically, we create a recommendation system by factorizing a large user-server matrix made up of all the users and servers in the enterprise, using NMF (non-negative matrix factorization) to predict what other servers this particular user is likely to access in future given his peer's actions. Another part of the second stage depends on servers being grouped together based on their functionality. This server clustering based model reduces false alarms from stage 1 PCA model by flagging users only if they access a new group of servers in the future. We present experimental results on the type of anomalous patterns we are able to uncover using this approach.

Room 102 - 3:05 PM to 4:05 PM
Elasticsearch, Spark, and Kafka for Near Real-Time Analytics in Financial Services

Financial Services (FS) organizations are under pressure to provide users with more timely access to data over rapidly expanding data sets using more sophisticated analysis techniques. As a result, FS companies are increasingly building data driven solutions that combine advanced analytical techniques (statistics, machine learning, and “deep” learning); architectures for near real-time ingestion, processing, and reporting; and “Big Data” technologies. In this talk, we present common requirements for these systems, details of how we solve for key issues, and common solution architectures. We will then detail a Proof of Value level solution built for a hypothetical insurance firm pricing property and casualty policies. In taking a case based approach, we highlight this firm’s business drivers and link technology design choices to outcomes of interest. The technology stack employed in the live demo consists of Kafka for real-time message brokering and queuing; Storm for data ingestion, HDFS for long term storage, Elastic search for near real-time data analysis and reporting, and Spark Streaming for near real-time data processing. Additionally, Spark MLlib and Spark batch processing are used for offline analysis / machine learning.

Room 101 - 3:05 PM to 4:05 PM
Oozie or Easy: Managing Hadoop Workflows the EASY Way

According to Gartner Inc., about 2/3 of all organizations are investing in Big Data and Hadoop but only 8% have production applications. Given the universal acceptance that Big Data is a huge competitive differentiator, it's likely that the companies that have not yet deployed Big Data applications are struggling to get their arms around this new technology. Industry experts agree that Hadoop and its ecosystem are extending the enterprise IT fabric rather replacing any major portions of it. This is particularly evident with workflow management where Hadoop jobs are growing in volume and complexity and are becoming more intertwined with many existing technologies. For IT organizations that already are overwhelmed by a proliferation of management tools, the ideal solution would be support for Hadoop's new workloads by the existing tools already in place, and more importantly, that already meet the required standards for governance, ease of use, reporting, planning and service level management. Such an approach will allow IT to leverage existing staff and mature best practices rather than having to re-invent the wheel yet again specifically for Hadoop.
